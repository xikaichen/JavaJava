1. 换行
非赋值运算符前断开 如 +， 他将位于下一行 同理 ||, && ,…
赋值运算符处断开 = 留在同一行
左括号留在同一行
逗号与其前面内容留在同一行
换行缩进至少4个空格

2.垂直空白
需要一个空行的地方：
类内的连续成员之间，字段，构造函数，方法，嵌套类。。。
允许多个连续空行，但不鼓励

3.水平空白
( {前面， 有连续两个括号例外
, : ; )后面
在一条语句后面注释 //两边都要有空格，允许多个 但没必要

4. 变量声明 一次只声明一个变量 不使用 int a, b;
需要时才声明，并尽快初始化

5. 注释
块注释与其周围的代码在同一缩进级别。它们可以是/* ... */风格，也可以是// ...风格。对于多行的/* ... */注释，后续行必须从*开始， 并且与前一行的*对齐。以下示例注释都是OK的。
/*
 * This is          // And so           /* Or you can
 * okay.            // is this.          * even do this. */
 */

6. 命名
类名都以UpperCamelCase风格编写。
方法名都以lowerCamelCase风格编写。通常是动词或动词短语。
常量名命名模式为CONSTANT_CASE
非常量字段名以lowerCamelCase风格编写。通常是名词或名词短语。
